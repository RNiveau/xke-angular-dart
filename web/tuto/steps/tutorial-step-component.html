Une des choses qui manque cruellement au developpement web moderne est la manque d’encapsulation. Le draft W3C sur les Web Components tente à resoudre notamment ce point. Le but de cette étape est de créer un web component qui encapsule le filtrage des logs (le bloc en haut permettant de filtrer les logs par HTTP Status, HTTP Méthod et Search). Ce composant ne communiquera avec le reste de l’application que par l’interface explicitement declarée et encapsulera le traitement, le markup mais aussi les styles.

Pour ce faire :

  • Definir la propriété query dans le LogController, la binder à l'input "Search a message" (index.html) et fixer le filter dans view-list.html
  • Créer la class 'NavigateComponent' dans le fichier xke-angular-dart/lib/workshop/navigate /navigate_component.dart
  • L'annoter avec @Component avec les paramètres suivantes : selector à 'navigate', templateUrl à 'packages/xke_angular_dart/workshop/navigate/navigate_component.html', cssUrl à 'packages/xke_angular_dart/workshop/navigate/navigate_component.css' et publishAs à 'cmp'
  • Dans cette class declarer les propriétés status, methods et query en les annotant avec @NgTwoWay
  • Déclarer le type NavigateComponent dans main.dart
  • Créer le fichier lib/workshop/navigate/navigate_component.css en y recopiant le contenu du xke-angular-dart/web/css/navigation.css
  • Créer le fichier lib/workshop/navigate/navigate_component.html et y déplacer l'integralité de l'élément #navigation (depuis index.html)
  • Changer le binding de logCtrl.* à cmp.* : en effet nous avons publié (publishAs) le scope du component en tant que cmp
  • Enfin, ajouter le component navigate dans index.html en bindant status, methods et query
Pour voir le component dans le DOM cocher la case "Show Shadow DOM" dans le settings de Developer tools de Chromium.
