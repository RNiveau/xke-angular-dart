Le filtre utilisé dans l'étape précédante et un filtre fourni par défaut dans AngularDart. Il est possible de créer ses propres filtres afin de pouvoir agir sur la collection d'éléments à afficher.

Les filtres personnalisés s'utilisent de la même façon que le précédant à travers la syntaxe | (pipe).

Cette étape consiste à créer deux filtres spécifiques aux logs. Un filtre sur les status et un autre sur les méthodes.

Pour ce faire, il faut créer les filtres de la même façon que le filtre 'TruncateFilter', déclarer leur type et appliquer les filtres sur le ng-repeat:

  • Déclarer un attribut status dans le contrôlleur 'LogController' ayant pour clé un verbe HTTP (200, 404, 500) et pour valeur un booléen à true (par défaut, tous les filtres sont sélectionnés)
  • Dans le fichier xke-angular-dart/web/index.html, binder à l'aide de l'attribut ng-model les différentes checkbox en faisant référence à la propriété de l'objet concernée (ex: logCtrl.status['200'] pour la checkbox concernant 200)
  • Dans le fichier xke-angular-dart/lib/workshop/filters.dart, créer un filtre 'StatusFilter' nommé 'statusFilter' ayant une méthode call prenant la collection en paramètre ainsi que l'état des checboxs de status
  • Déclarer le type StatusFilter dans le 'WorkshopModule'
  • Appliquer le filtre StatusFilter dans le ng-repeat
  • Déclarer un attribut methods dans le contrôlleur 'LogController' ayant pour clé un status HTTP (GET, POST, PUT, DELETE) et pour valeur un booléen à true (par défaut, tous les filtres sont sélectionnés)
  • Dans le fichier xke-angular-dart/web/index.html, binder à l'aide de l'attribut ng-model les différentes checkbox en faisant référence à la propriété de l'objet concernée (ex: logCtrl.methods['GET'] pour la checkbox concernant GET)
  • Dans le fichier xke-angular-dart/lib/workshop/filters.dart, créer un filtre 'MethodFilter' nommé 'methodFilter' ayant une méthode call prenant la collection en paramètre ainsi que l'état des checboxs de méthodes
  • Déclarer le type MethodFilter dans le 'WorkshopModule'
  • Appliquer les filtre MethodFilter dans le ng-repeat
