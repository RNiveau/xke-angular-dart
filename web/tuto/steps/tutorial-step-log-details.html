Un routeur qui gère une seule route, c'est dommage ! L'objectif de cette étape est de gérer une seconde route (avec un second contrôleur et un second template) pour afficher les détails d'un log.

Pour ce faire :

  • Dans le constructeur du WorkshopModule, ajouter le code suivant: factory(NgRoutingUsePushState, (_) =>
    new NgRoutingUsePushState.value(false));
  • Dans le fichier xke-angular-dart/lib/workshop/router.dart, déclarer une nouvelle route detail au niveau du routeur qui match l'URL /detail/:detailId avec le template detail.html
  • Créer un contrôleur DetailController exposé comme detailCtrl et ayant pour directive detail-ctrl. Le constructeur de ce contrôlleur prend en paramètre un service Http et un RouteProvider
  • Le 'DetailController' expose le log sélectionné dans l'attribut log
  • Dans le fichier xke-angular-dart/web/view-list.html, rajouter sur les id des logs un lien pointant vers la nouvelle route.
  • Dans le fichier xke-angular-dart/web/detail.html, afficher dans le tableau les différentes informations du log courant (id, date, url, method, status, message)
