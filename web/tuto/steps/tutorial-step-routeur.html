Le framework AngularDart a pour cible le développement d'application dîtes "single-page application". En d'autres termes, tout ou partie des fonctionnalités d'une application sont exposées au sein d'une même page.

Néanmoins, il est important d'exposer des URLs distinctes pour conserver un historique de navigation et être capable d'afficher des informations dépendantes du contexte de l'URL.

Le routeur est un composant AngularDart qui permet de faire le mapping entre une URL (avec potentiellement des paramètres dynamiques), un contrôleur et un template. Cela permet au framework d'émuler une navigation à l'aide de liens préfixés avec un # (ex /#detail/1 pour afficher le log dont l'ID est 1). L'avantage est que ces URLs sont bookmarkables et donc accessibles de l'extérieur sans avoir eu besoin d'effectuer un quelconque parcours sur le site Web.

Pour initialiser le routeur, AngularDart fait appel à une méthode RouteInitializerFn définie pour chaque application et indiquée à AngularDart à travers la méthode value

Cette étape a pour objectif la mise en place d'un routeur qui gérera l'accès à la racine du site Web et qui pointerait sur un template dédié à l'affichage des logs. Pour ce faire :

  • Dans le fichier xke-angular-dart/lib/workshop/router.dart, définir la méthode routeInitializer répondant au prototype RouteInitializerFn et créer une route / faisant référence au template view-list.html
  • Dans le fichier xke-angular-dart/web/index.html, rajouter la propriété ng-view sous la div #navigation
  • Dans le fichier xke-angular-dart/web/view-list.html, externaliser le contenu du fichier xke-angular-dart/web/index.html permettant l'affichage des logs
