La philosophie d'AngularJS n'est pas de manipuler le DOM directement, comme on peut en avoir l'habitude avec des frameworks type jQuery.

Pour interagir avec le DOM, AngularJS propose d'utiliser un composant appelé directive.

L'objectif de cette étape est d'afficher un message lorsqu'aucun log ne match les filtres sélectionnés.

  • Dans le fichier workshop-angular/app/js/app.js, déclarer une directive appelée toggleVisibility au sein du module. La directive retourne une fonction qui prend en paramètre le scope, une référence à l'élément HTML au sein duquel la directive sera déclarée et un troisième paramètre qui donnera accès aux propriétés de la directive
  • Pour l'implémentation de cette fonction, invoquer la fonction $watch du $scope en lui passant l'attribut toggleVisibility du troisième paramètre puis une fonction callback
  • Implementer la fonction callback pour que l'affichage de l'element HTML soit activé lorsque l'expression passée en paramètre du callback vaut true
  • Dans le fichier workshop-angular/app/views/log-list.html, définir un message indiquant qu'aucun log ne s'affiche dans une balise utilisant la nouvelle directive et passer en paramètre de cette directive une expression qui renvoie true lorsque le nombre de logs (selectedLogs) affichés est égal à 0
