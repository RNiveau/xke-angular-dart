Le contrôleur est l'un des composants fondamentaux du framework AngularDart. Il s'agit d'une classe Dart spéciale qui permet de faire le lien entre le model et la vue selon les principes du MVC.

L'annotation @Controller permet de définir une classe comme Controller

L'objectif de cette étape est donc la création d'un contrôleur et l'affichage d'un élément de son scope dans la page (en l'occurence les logs) :

  • Dans le fichier xke-angular-dart/lib/workshop/log_controller.dart, Créer la classe LogController (vous remarquerez que l'annotation Controller est dépréciée, c'est normal ...)
  • Définir la classe comme controller en le publiant en tant que logCtrl et le sélecteur log-ctrl
  • Définir l'attribut logs au sein du LogController et appeler le service MockServiceLog.getLogs() dans le constructeur pour lui donner une valeur (pour le moment, les logs sont mis en dur dans le contrôleur, par la suite ils seront récupérés depuis un backend)
  • Déclarer le type LogController dans le constructeur du WorkshopModule
  • Dans le fichier xke-angular-dart/web/index.html, ajouter l'attribut log-ctrl à la div #angular-app, puis afficher le contenu de la variable logCtrl.logs dans la page
